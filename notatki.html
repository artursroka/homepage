Tydzień 4 życie w pudełku

display: block
Sposób na wyświetlanie elementów jeden pod drugim. Ta własność powoduję, że element zajmuje całą szerokość rodzica.

display: inline
Dzieki tej własności elementy zachowuję się jak tekst. Układa się jeden po drugim i przechodzi do następnego wersu jak nie mieści się. Ignoruje podene wymiary obiektu.

display: inline-block
Łączy właśności inline i block, to zanaczy elementy układają się jak inline, ale można im nadać szerokość i wysokość (width / height).

overflow: hidden
Wartość ta odcina zawartość, która nie mieści się w obiekcie.

overflow: visible
Domyślna wartość. Powoduje, że w obiecie mniejszym, większy jest też widoczny.

overflow: auto
Jeżeli dany element nie mieści się, dodaje scrola

git init
Polecenie zakłada repozytorium (projekt). W wierszu poleceniń należy wejść w katalog z projektem i użyć komendy

Commit = stan projektu
Comitujemy projekt po wprowadzeniu zmian. dodajemy krótki komentarz co zmieniliśmy

git status
pokazuje status projektu. Czyli które pliki są śledzone i które są zmienione.

git add .
Dodaje pliki. Polecenia należy użyć bedąć w katalogu którym znajduj się projekt.

git commit -m "opis zmian"
w ten sposób comitujemy w sposób któtki i treściwy

git push
wypuszczamy pliki do repozzytorium w githubie
